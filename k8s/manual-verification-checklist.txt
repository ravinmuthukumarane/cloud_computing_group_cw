Manual Kubernetes verification checklist
========================================

Cluster evidence
----------------

Command:

kubectl get nodes

Expected:

One k3d node should be Ready.


Namespace evidence
------------------

Command:

kubectl get namespaces

Expected:

app and data namespaces should exist.


Data layer evidence
-------------------

Command:

kubectl get pods -n data
kubectl get pvc -n data
kubectl get service -n data

Expected:

postgres pod Running.
postgres-pvc Bound.
postgres service available as ClusterIP.


App layer evidence
------------------

Command:

kubectl get pods -n app
kubectl get deployments -n app
kubectl get services -n app

Expected:

frontend, bff, identity, salary-submission, vote, search, and stats should be
Running and Ready.


Ingress evidence
----------------

Command:

kubectl get ingress -n app

Expected:

app-ingress should exist.


Logs evidence
-------------

Command:

kubectl logs -n data deployment/postgres
kubectl logs -n app deployment/bff
kubectl logs -n app deployment/frontend

Expected:

Postgres should show startup logs.
BFF should show service startup.
Frontend should show nginx access/startup logs.


Postgres schema evidence
------------------------

Command:

kubectl exec -n data deployment/postgres -- psql -U postgres -d cloud_salary -c "\dn"

Expected:

Schemas should include identity, salary, and community.

Command:

kubectl exec -n data deployment/postgres -- psql -U postgres -d cloud_salary -c "\dt identity.*"
kubectl exec -n data deployment/postgres -- psql -U postgres -d cloud_salary -c "\dt salary.*"
kubectl exec -n data deployment/postgres -- psql -U postgres -d cloud_salary -c "\dt community.*"

Expected:

Required tables should exist.


Health evidence
---------------

Command:

curl http://localhost:8080/api/health

Expected:

{"status":"ok","service":"bff"}


End-to-end workflow evidence
----------------------------

1. Open frontend:

http://VM_PUBLIC_IP:8080/

2. Submit anonymous salary information.

3. Sign up a user.

4. Log in and receive a token.

5. Vote on a salary submission.

6. Repeat voting until threshold is reached if needed.

7. Confirm the submission becomes approved.

8. Search for approved salary records.

9. Confirm stats include approved salary data.
